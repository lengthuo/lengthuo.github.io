定义-代码规范

命名规范

  1. 命名均不能以下划线开始。
  2. 命名不能使用拼音与英文混合的方式或者拼音方式
  3. 命名不能使用不规范的缩写
  4. 类命名使用大驼峰风格，如果遇到简写，简写第一个字符大写，其余小写
  5. 函数名、变量名、方法名、参数名、成员变量名使用小驼峰风格
  6. 常量名使用大写，单词间用下划线隔开
  7. 异常类命名使用 Exception 结尾，测试类命名以它要测试的类的名称开始，以 Test 结尾。队列类以Queue结尾；事件类以Event结尾。枚举类名带上 Enum 后缀，枚举成员名称需要全大写，单词间用下划线隔开
  8. Service/Model 层方法命名规约
  9. 获取单个对象的方法用 get 做前缀。
  10. 获取多个对象的方法用 list 做前缀。
  11. 获取统计值的方法用 count 做前缀。
  12. 插入的方法用 save 做前缀。
  13. 删除的方法用 remove 做前缀。
  14. 修改的方法用 update 做前缀。

编码规范

使用 不带BOM的UTF-8 编码

布局规范

  1. 每个 namespace 命名空间声明语句和 use 声明语句块后面，必须 插入一个空白行。
  2. 类的开始花括号 { 必须写在函数声明后自成一行，结束花括号 } 也必须写在函数主体后自成一行。
  3. 方法的开始花括号 { 必须 写在函数声明后自成一行，结束花括号 } 也必须写在函数主体后自成一行。
  4. 类的属性和方法 abstract 以及 final 必须 声明在访问修饰符之前，而 static 必须 声明在访问修饰符之后
  5. 类最多包含20个方法，一个类长度最多400行
  6. 函数/方法最多有50行代码
  7. 函数/方法有默认值的参数必须放在最后
  8. 原生及引入类规范 不能使用过时的类或方法或扩展
