常用命令

没有先后顺序，最近用到的，想起来就做下笔记。

  • telnet 127.0.0.1 3301 检测某ip对应的端口是否可通

  • ssh d1@123.206.62.146 -p 60022 远程连接

  • tail -f xxx文件 监听文件

  • tail -n +10 // 从10行开始显示到结束的内容

  • tail -n 10 // 显示最后10 行的内容

  • head -n 10 // 显示开头 10 行的内容 head -n +10 // 同 head -n 10

查看系统编码

查看支持的字符编码 locale

修改/etc/sysconfig/i18n，如改成中文编码： LANG=C.UTF-8 （这个编码是当时金融云走网关出现乱码的时候。）

把程序放到后台执行 恢复


需要注意，放入后台的命令必须和用户没有交互的，
例如 vi xx  & 编辑一个文件是个用户有交互的 即使使用& 也同样是暂停的
 
php artisan serve &  后台执行
ctrl + z   暂停快捷键 放入后台是暂停的


vi a &
php artisan serve &
php artisan serve --port = 8001   ctrl+z

显示：
[1]  - suspended (tty output)  vi a
[2]    running    php artisan serve
[3]  + suspended  php artisan serve --port=8001

jobs -l  （-l 显示进程id） + 号表示最后一个放入的 -号表示倒数第二个放入的 ：
[1]  - 46579 suspended (tty output)  vi a
[2]    46730 running    php artisan serve
[3]  + 46875 suspended  php artisan serve --port=8001


恢复

fg %工作号    工作号是1|2|3 而不是进程号   
bg %后台执行

后台命令脱离终端执行方法

  1. 把需要执行的文件放到/etc/rc.local文件 （每次启动都会启动该文件）
  2. 使用系统定时任务，让系统在制定的时间执行某个后台命令
  3. nohup 命令

ps 查看进程


ps aux 查看系统中的所有进程，BSD操作系统格式

ps -le 查看系统中的所有进程，使用linux标准命令格式

 
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root         1  0.0  0.0  43176  3304 ?        Ss   Jun25 128:29 /usr/lib/systemd/systemd --switch
root         2  0.0  0.0      0     0 ?        S    Jun25   0:01 [kthreadd]

USER运行者 
PID进程号 
cpu 
MEM物理内存 
VSZ虚拟内存KB 
RSS真是内存KB 
STAT终端号  1-6 个终端 7是图像界面终端   pts/0-255 代表远程虚拟终端   ？号说明都不是通过终端来启动的而是由内核直接启动
START启动时间
TIME 耗费cpu的运行资源


stat 中的参数意义如下：
   D 不可中断 Uninterruptible（usually IO）
   R 正在运行，或在队列中的进程
   S 处于休眠状态
   T 停止或被追踪
   Z 僵尸进程
   W 进入内存交换（从内核2.6开始无效）
   X   死掉的进程

    < 高优先级
    n   低优先级
    s   包含子进程
    +   位于后台的进程组

pstree 查看进程数（进程关系）


 -p : 显示进程ID
 -u : 显示进程所属用户

rz sz

rz 本地上传文件到服务器（ssh）
sz 选定的文件发送到服务器
