进程线程

进程(process):多进程的好处是，一个进程崩溃不会影响其他进程，

进程是一组有自己的内存和执行路径的计算机指令；

线程（thread):

线程虽然是一组计算机指令，但线程都执行在一个宿主的执行路径里，没有自己的内存。

多进程的好处是：一个进程崩溃不会影响其他进程，
多线程的好处是：线程更容易创建于管理,没有内存分配/合并方面的开销，线程的切换非常迅速，因为无需切换上下文

应用

如论学什么东西都要尽可能的知道一些底层的东西，有利于以后的使用。

java

是由C++ 开发的。单进程，多线程。，索引在开发时候，一般都的考虑多线程的问题,例如简单的单例书写，由于是多进程，所以操作单例的时候一定要加锁。来保证单例，而php就不用，因为php是单进程单线
程的。所以不会产生同时的问题。

php

由于是单进程单线的，所以开发会相对简单，不会设计太多的锁问题。所以php会更快上手。

mysql

单进程多线程

多线程的。也就是说，所有的程序中的链接都只是一个线程，他们都在一个进程里面。下面我举一个例子：

当2个事务同时更新同一个表的同一条记录的时候，在都没有commit之前，会有一个发生等待，为什么会等待。mysql 是CS结构的 （ 客户--client--server物理数据。 服务器--mysql-server是服务端），会有一个主进程处理所有的请求。在主进程中会监控所有子进程的所有的操作包括事务的操作。这时候主进程监测到，之后会按照顺序执行，下一个事务就等待。

网上常常会看到这个问题：为什么Oracle在Windows上是多线程，而在Linux上是多进程的？
传统的unix系统，早期没有提供多线程，只有多进程。linux是最近的版本才加入多线程支持，以前一直都是多进程。windows很早就支持多线程，本地应用大部分也是多线程。因此oracle在windows上一直都是多线程，在unix上才是多进程。所以mysql很可能是从windows发站起来的。所以mysql一开始就是多线程的。

redis

Redis 为什么使用单进程单线程方式也这么快
Redis 采用的是基于内存的采用的是单进程单线程模型的 KV 数据库，由 C 语言编写。官方提供的数据是可以达到100000+的 qps。这个数据不比采用单进程多线程的同样基于内存的 KV 数据库 Memcached 差。

Redis 快的主要原因有：

完全基于内存；
数据结构简单，对数据操作也简单；
使用多路 I/O 复用模型；

nginx

nginx使用一个多进程模型来对外提供服务，其中一个master进程，多个worker进程。master进程负责管理nginx本身和其他worker进程。
nginx平台初探(100%)

单进程单线程好处

代码更清晰，处理逻辑更简单
不用去考虑各种锁的问题，不存在加锁释放锁操作，没有因为可能出现死锁而导致的性能消耗
不存在多进程或者多线程导致的切换而消耗 CPU

单进程单线程弊端

无法发挥多核 CPU 性能，不过可以通过在单机开多个 Redis 实例来完善；
