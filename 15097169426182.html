笔记-git 常用命令

Git中已被跟踪的文件怎样在git .gitignore 配置

1.首先把这个文件从版本库里移除：Git rm --cached file (Git rm --cached -r folder ) 删除文件夹用括号中的命令， 再把这个文件或文件夹加入到.gitignore文件中。
2.其次进行提交：git commit -m 'comment'
3.最后进行推送：git push

git打标签

git tag -d Release_1_ 删除标签
git tag -a v0.1 -m "version 0.1 released" 3628164
查看标签 git tag
git show v0.1 可以查看说明信息

默认标签是打在最新提交的commit上的。所以跟分支没有关系。

git 默认不区分文件名大小写

一  配置git 使其对文件名大小写敏感
git config core.ignorecase false
二 从git 本地仓库删除此文件,然后添加再提交
(1) 删除 git rm readme.md
(2) 重新添加 git add Readme.md
(3)提交 git commit -m 'Readme.md'

git reset 将当前HEAD复位到指定状态


git reset HEAD~1            保留修改，不保留commit状态
commit会全部撤退，但是修改的内容都在  
  
git reset --soft HEAD~1     
场景 ：（很有用），100 个文件修改 10个不想push，90个想push，
解决方案：1 先reset --soft xxx （主分支最新的地方），只回退commit，文件所有的我修改都会保留

git reset --hard HEAD~1     
不保留修改，直接跳转到上个commit的状态(清空所有的修改，所有的修改都清空，注意)
所谓不保留：是指所有staged 的内容，全部清空，没有add的内容不会清空(modified 状态的内容会全部修改)

总结三个：

在服务器上回退版本：
 git reset --hard 9ab191e7bf709299831f5aae290af5a104b2fa3f
 服务器上回退代码是不适用最新提价的代码，所以必须hard 如果使用最上面的2中，代码还是会保留的。
 
在本地回退最好使用最上面的2中：目的为了保留自己的修改

设置提交人信息

// 设置全局
git config --global user.name "Author Name"
git config --global user.email "Author Email"

// 或者设置本地项目库配置
git config user.name "Author Name"
git config user.email "Author Email"

修改最后一次提交信息


Git commit –amend  "xxxx"

修改文件权限禁止git捕获


git config core.filemode false

仅从git仓库中删除指定文件，而保留其在working tree中

git rm --cached <文件名> # 仅从git仓库中删除<文件名>而在working tree中保留

git历史 展示

git log --oneline --graph --decorate --all

多次commit合并成


git rebase -i  id  （合并这个id之前的）

其中，-i 的参数是不需要合并的 commit 的 hash 值.


当前我们只要知道 pick 和 squash 这两个命令即可。

1. pick 的意思是要会执行这个 commit
2. squash 的意思是这个 commit 会被合并到前一个commit

 保存退出之后就会弹出commit message 的编辑界面
 编辑commit的message即可

git log --oneline --graph --decorate --all

神奇的发现变了

git pull --rebase(变基)

git pull = git fetch + git merge
git pull --rebase = git fetch + git rebase

git fetch = 从远程获取最新版本到本地，不会自动merge
git merge = 合并
（git fetch更安全一些因为在merge前，我们可以查看更新情况，然后再决定是否合并）
git rebase = 老的commit丢弃，下图中c5 c6会被丢弃（变基）
立即变基之后，你就会知道为什么，git rebase 一定要保持你本地不能有commit的文件


理解上面的就可以理解这句
git pull --rebase,这里表示把你的本地当前分支里的每个提交(commit)取消掉，并且把它们临时 保存为补丁(patch)(这些补丁放到".git/rebase"目录中),然后把本地当前分支更新 为最新的"origin"分支，最后把保存的这些补丁应用到本地当前分支上。

pull --rebase 的时候要不你就stash或者commit（commit之后会按照上面的给你处理掉）

git stash (栈存当前修改)

stash 存储当前修改（是用栈来存储当前代码），
git stash --help
git stash save 
git stash pop (是栈的形式存储的，所以pop的时候直接取出，而且在容器里面会删除这条记录)

应用-创建分支，提交远程分支，关联远程分支

  1. git checkout -b dev （创建分支并切换，然后直接git push -u 会有提示。）
  2. git branch --set-upstream-to=origin/dev （可忽略）
  3. git push --set-upstream origin dev （本地分支推送到远程）
  4. git push -u
