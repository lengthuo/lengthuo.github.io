定义-返回值格式

返回格式

{
    "isException":true|false
    "code":xxx
    "msg":xxx
    "data":
}

代码处理

<?php
/**
 * @author lishuo
 */

namespace App\Tools;


use Illuminate\Database\Eloquent\Collection;
use Illuminate\Database\Eloquent\Model;
use Illuminate\Pagination\LengthAwarePaginator;
use Illuminate\Support\Facades\Response;

/**
 * 返回格式化
 * Class ReturnFormat
 * @package App\Tools
 */
class ReturnFormat
{
    /**
     * @param array $data
     * @param string $state
     * @param string $message
     * @param $isException
     * @return mixed
     */
    public static function message($data = [], $state = 'SUCCESS', $message = '', $isException = false)
    {
        self::formatData($data);
        $data = !is_array($data) ? [] : $data;//返回值为true的时候直接返回空数组
        $code = self::getCodeByStates($state);
        $message = $message ? : self::getMessageByStates($state);
        return Response::json([
            'isException' => $isException,
            "code"        => $code,
            "msg"         => $message,
            "data"        => $data,
        ]);
    }

    /**
     * 数据格式化  处理分页，具体分页参考分页的标准定义
     * @param $data
     */
    private static function formatData(&$data)
    {
        if ($data instanceof LengthAwarePaginator) {
            $data = $data->toArray();
            $data['list'] = $data['data'];
            unset($data['data'], $data['next_page_url'], $data['path'], $data['prev_page_url'], $data['last_page'], $data['from'], $data['to']);
        }
    }

    private static function getCodeByStates($state)
    {
        return isset(config("code." . $state)['code']) ? config("code." . $state)['code'] : '500.';
    }

    private static function getMessageByStates($state)
    {
        return isset(config("code." . $state)['msg']) ? config("code." . $state)['msg'] : '未知错误';
    }
}

定义-git提交代码规范

实例

feat(module):(JRY-120) 后台订单列表

定义


<type>(<scope>):<jira:id><subject>
    <BLANK LINE>
<body>
   <BLANK LINE>
<footer> 

jira:id : 
       JIRA 号(没有填:JRY-00)
type：
       下面有几个我们常用到的
      . feat :新功能 
      . fix :修复bug  
      . doc : 文档改变
      . style : 代码格式改变
      . refactor :某个已有功能重构
      . perf :性能优化
      . test :增加测试
      . build :改变了build工具 如 grunt换成了 npm
      . revert: 撤销上一次的 commit 
      . chore 维护
scope：
       用来说明此次修改的影响范围 可以随便填写任何东西，推荐使用
       all ：表示影响面大 ，如修改了网络框架  会对真个程序产生影响
       loation： 表示影响小，某个小小的功能
       module：表示会影响某个模块 如登录模块、首页模块 、用户管理模块等等  
       
subject:
       用来简要描述本次改动，概述就好了
body:
       具体的修改信息 应该尽量详细
footer：
       放置写备注啥的。

定义-中心返回错误到p2p

当p2p调用中心时候，如果返回结果非1000 或者是 不是业务相关的返回值，都需要把信息返回给接入方

例如：
p2p 调用中心的时候返回：

    array:4 [
        "isException" => true
        "code" => 404
        "msg" => "请求的资源不存在"
        "data" => []
    ]

通过配置文件code（code文件是一致的），找出对应信息，然后抛出给接入方

    if($res[ 'code' ] != 1000) {
        throw new CenterException(config('code.'.$res[ 'code' ]));
    }

如果返回2001 是业务代码的话 就是返回给p2p的，如果p2p需要处理就处理，不需要处理同样可以直接抛给接入方，根据业务来决定
