定义-异常处理

修改laravel异常类handler.php
异常返回统一处理


<?php

namespace App\Exceptions;

use App\Tools\LogWrite\LogWrite;
use App\Tools\ReturnFormat;
use Exception;
use Illuminate\Auth\AuthenticationException;
use Illuminate\Foundation\Exceptions\Handler as ExceptionHandler;

class Handler extends ExceptionHandler
{
    /**
     * A list of the exception types that should not be reported.
     *
     * @var array
     */
    protected $dontReport = [
        \Illuminate\Auth\AuthenticationException::class,
        \Illuminate\Auth\Access\AuthorizationException::class,
        \Symfony\Component\HttpKernel\Exception\HttpException::class,
        \Illuminate\Database\Eloquent\ModelNotFoundException::class,
        \Illuminate\Session\TokenMismatchException::class,
        \Illuminate\Validation\ValidationException::class,
    ];

    /**
     * Report or log an exception.
     *
     * This is a great spot to send exceptions to Sentry, Bugsnag, etc.
     *
     * @param  \Exception  $exception
     * @return void
     */
    public function report(Exception $exception)
    {
        parent::report($exception);
    }

    /**
     * Render an exception into an HTTP response.
     *
     * @param  \Illuminate\Http\Request  $request
     * @param  \Exception  $exception
     * @return \Illuminate\Http\Response
     */
    public function render($request, Exception $exception)
    {
        $message = $this->parseException($exception);
        $errors = [
            "error_file" => $exception->getFile(),
            "error_code" => $exception->getCode(),
            'error_message' => $exception->getMessage(),
        ];
        LogWrite::error(array_merge($message, $errors), "异常错误",'error');
        return ReturnFormat::message([], $message['status'], $message['msg'], true);//统一返回参数不合
    }

    /**
     * Convert an authentication exception into an unauthenticated response.
     *
     * @param  \Illuminate\Http\Request  $request
     * @param  \Illuminate\Auth\AuthenticationException  $exception
     * @return \Illuminate\Http\Response
     */
    protected function unauthenticated($request, AuthenticationException $exception)
    {
        if ($request->expectsJson()) {
            return response()->json(['error' => 'Unauthenticated.'], 401);
        }

        return redirect()->guest('login');
    }
    /**
     * 解析失败
     * @return array
     */
    protected function parseException(Exception $exception)
    {
        $message = 'TNTERNAL_SERVER_ERROR';
        $msg = '';
        if ($exception instanceof LoginException || xxx 捕捉自动以异常
        ) {
            $message = $exception->getMessage();
        } elseif ($exception instanceof ConnectionException) {
            $message = 'REDIS_HANDLING_ERROR';
        } elseif ($exception instanceof NotFoundHttpException) {
            $message = 'NOT_FOUND';
        } elseif ($exception instanceof MethodNotAllowedHttpException) {
            $message = 'METHOD_NOT_ALLOWED';
        } elseif ($exception instanceof ValidationException) {
            $errors = $exception->validator->errors()->getMessages();
            foreach ($errors as $k => $v) {
                $msg = $k . '|' . $v[0];
                break;
            }
            $message = 'PARAM_INVALID';
        } elseif ($exception instanceof QueryException) {
            $message = 'DATA_HANDLING_ERROR';
        }
        LogWrite::exception($exception);
        return ['status' => $message, 'msg' =>$msg ];
    }
}

文档-单元测试

写单元测试的目的

  1. 帮助开发人员识别并修复bug、重构代码
  2. 逼迫我们去想业务逻辑，使开发速度慢下来，质量提上去

测试工具

phpUnit、mockery，两个工具 Laravel框架 自带，也可以自己在本地安装phpUnit。

mockery 官方文档：http://docs.mockery.io/en/latest/reference/startup_methods.html （英文）

phpUnit 官方文档：https://phpunit.de/manual/current/zh_cn/phpunit-book.html （中文）

https://phpunit.de/manual/current/zh_cn/index.html

mock

测试的时候最好不要真正的去连接数据库，依赖数据库，不利于维护，所以尽量使用mock

代码示例


use Mockery as m;

class OrderCreateServiceTest extends TestCase
{

    public function setUp()
        {
            parent::setUp();  //必须先执行这一句，才能调用laravel提供的东西（初始化容器）
            $this->mockAccountRepositories = m::mock('App\Repositories\AccountRepositories');
            $this->mockProductRepositories = m::mock('App\Repositories\ProductRepositories');
            $this->mockOrderRepositories = m::mock('App\Repositories\OrdersRepositories');
            $this->mockRequest = new Request();
            $this->mockCanBuyValidateService = new CanBuyValidateService();
            $this->orderCreateService = new OrderCreateService(
                $this->mockAccountRepositories, $this->mockProductRepositories,
                $this->mockOrderRepositories, $this->mockCanBuyValidateService);
        }
        
//提供多种方式，但规定必须以这种方式定义  (开发中规定最大，保持团队的统一)
 public function test_should_fail_if_account_info_is_empty()
    {
        $this->mockRequest->userid = 1;
        //mock的写法
        $this->mockAccountRepositories->shouldReceive('getInfoByUid')
            ->with($this->mockRequest->userid)
            ->andReturn([]);
        $this->expectException('App\Exceptions\StatesException');
        $this->orderCreateService->handle($this->mockRequest);

    }
}

断言

布尔类型
  1. assertTrue 断言为真
  2. assertFalse 断言为假
NULL类型
  1. assertNull 断言为NULL
  2. assertNotNull 断言非NULL
数字类型
  1. assertEquals 断言等于
  2. assertNotEquals 断言不等于
  3. assertGreaterThan 断言大于
  4. assertGreaterThanOrEqual 断言大于等于
  5. assertLessThan 断言小于
  6. assertLessThanOrEqual 断言小于等于
字符类型
  1. assertEquals 断言等于
  2. assertNotEquals 断言不等于
  3. assertContains 断言包含
  4. assertNotContains 断言不包含
  5. assertContainsOnly 断言只包含
  6. assertNotContainsOnly 断言不只包含
数组类型
  1. assertEquals 断言等于
  2. assertNotEquals 断言不等于
  3. assertArrayHasKey 断言有键
  4. assertArrayNotHasKey 断言没有键
  5. assertContains 断言包含
  6. assertNotContains 断言不包含
  7. assertContainsOnly 断言只包含
  8. assertNotContainsOnly 断言不只包含
对象类型
  1. assertAttributeContains 断言属性包含
  2. assertAttributeContainsOnly 断言属性只包含
  3. assertAttributeEquals 断言属性等于
  4. assertAttributeGreaterThan 断言属性大于
  5. assertAttributeGreaterThanOrEqual 断言属性大于等于
  6. assertAttributeLessThan 断言属性小于
  7. assertAttributeLessThanOrEqual 断言属性小于等于
  8. assertAttributeNotContains 断言不包含
  9. assertAttributeNotContainsOnly 断言属性不只包含
  10. assertAttributeNotEquals 断言属性不等于
  11. assertAttributeNotSame 断言属性不相同
  12. assertAttributeSame 断言属性相同
  13. assertSame 断言类型和值都相同
  14. assertNotSame 断言类型或值不相同
  15. assertObjectHasAttribute 断言对象有某属性
  16. assertObjectNotHasAttribute 断言对象没有某属性
class类型
  1. class类型包含对象类型的所有断言，还有
  2. assertClassHasAttribute 断言类有某属性
  3. assertClassHasStaticAttribute 断言类有某静态属性
  4. assertClassNotHasAttribute 断言类没有某属性
  5. assertClassNotHasStaticAttribute 断言类没有某静态属性
文件相关
  1. assertFileEquals 断言文件内容等于
  2. assertFileExists 断言文件存在
  3. assertFileNotEquals 断言文件内容不等于
  4. assertFileNotExists 断言文件不存在
XML相关
  1. assertXmlFileEqualsXmlFile 断言XML文件内容相等
  2. assertXmlFileNotEqualsXmlFile 断言XML文件内容不相等
  3. assertXmlStringEqualsXmlFile 断言XML字符串等于XML文件内容
  4. assertXmlStringEqualsXmlString 断言XML字符串相等
  5. assertXmlStringNotEqualsXmlFile 断言XML字符串不等于XML文件内容
  6. assertXmlStringNotEqualsXmlString 断言XML字符串不相等
